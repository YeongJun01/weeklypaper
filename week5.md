# 1. 🇶 프로세스와 스레드에 대해 설명해 주세요.

- 프로세스는 독립적인 메모리 공간과 자원을 가진 독립적인실행 단위이고, 스레드는 하나의 프로세스 내에서 자원을 공유하며 실행되는 더 작은 단위의 실행 흐름이다.
  <br/>
  |구분|프로세스|스레드|
  |:---:|:---:|:---:|
  |정의|운영체제로분터 자원을 할당받는 작업 단위|프로세스 내에서 실행되는 실행 흐름의 단위|
  |메모리|독립된 메모리 공간을 가짐|같은 프로세스 내의 다른 스레드와 메모리 공간을 공유|
  |자원|자체 시스템 자원(코드, 데이터, 힘 등)을 가짐|프로세스의 코드, 데이터, 힙 영역을 공유하고, 스택 영역만 별도로 가짐|
  |통신|푸로새스 간 정보 공유를 위해 IPC가 필요|프로세스 내 같은 메모리 공간을 공유하므로 데이터 공유가 용이|
  |생성/전환|생성에 비용이 많이 들고, 문맥 교환(Context Switching) 비용이 높음|생성 및 문맥 교환 비용이 적어 더 효율적|
  |특징|하나의 프로세스가 충돌해도 다른 프로세스에 영향을 주지 않아 안정적|한 스레드의 오류가 같은 프로세스 내의 다른 스레드에 영향을 줄 수 있어 동기화에 주의해야함|
  <br/>

# 2. 🇶 프레임워크와 라이브러리의 차이에 대해 설명해 주세요.

- 프레임워크 : 개발자가 소프트웨어를 개발함에 있어 코드를 구현하는 개발 시간을 줄이고, 코드의 재사용성을 증가시키기 위해 일련의 클래스 묶음이나 뼈대, 틀을 라이브러리 형태로 제공되는 것을 말한다.
- 특징

  - 개발자가 따라야 하는 가이드를 제공
  - 갭라할 수 있는 범위가 정해져 있음
  - 개발자를 위한 다양한 도구, 플러그인 지원

- 라이브러리 : 라이브러리란 개발자가 만든 클래스들의 나열로, 다른 프로그램들에서 사용할 수 있도록 제공하는 방식
- 라이브러리와 프레임워크의 차이는 제어 흐름에 대한 주도성이 누구에게 / 어디에게 있는가에 있다. 즉, 애플리케이션의 흐름을 누가 쥐고 있느냐에 달려있다.
  - 프레임워크는 프로그래밍에 대한 규칙을 가지고 있고 이 규칙을 따라야한다. 하지만 라이브러리는 개발자가 필요한 시점에 호출해 능동적으로 사용이 가능하다.
