### 🇶 세션 기반 인증에 대해 설명해 주세요.

- 서버 메모리에 정보를 저장하고 세션 아이디만 유저에게 전달한다.
- 유저의 브라우저에 있는 쿠키에 받은 세션 아이디를 저장한다.
- 유저가 요청을 보낼 때 쿠키에 있는 세션 아이디를 같이 서버에 보낸다.
- 서버는 메모리에 있는 정보에서 일치하는 세션 아이디를 찾아서 응답한다.
- 장점: 서버에 저장하기 떄문에 관리 효율
- 단점: 페이지 이동마다 db에 요청하므로 db 과부하가 걸린다.

### 🇶 토큰 기반 인증에 대해 설명해 주세요.

- 요청 시 유저 정보를 토대로 토큰을 만든다. -> 만료 일자 설정
- 유저는 쿠키를 통해 해당 토큰을 받는다.
- 유저다 여러 페이지를 이동 시 토큰을 보내주면 해당 서버에서 토큰 복호화(해석) 진행 => 해당 토큰과 키 값을 이용해 데이터 추출
- 장점: 메모리 부담 없다.
- 단범: 쿠팡 사태 -> 암호화를 풀 수 있다면 해킹의 위험이 있음
